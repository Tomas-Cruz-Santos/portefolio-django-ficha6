Componentes
Model

Define a estrutura dos dados em models.py. Cada classe corresponde a uma tabela SQL gerida pelo ORM do Django, que evita escrever SQL manualmente.

View

Contém a lógica de negócio em views.py. Recebe pedidos HTTP, consulta os Models, e devolve uma resposta com o Template renderizado.

Template

Ficheiros HTML com a linguagem de template Django. Apresentam os dados ao utilizador usando {% verbatim %}{{ variavel }}{% endverbatim %} e tags como {% verbatim %}{% for %}{% endverbatim %}.

2. Modelação

Diagrama Entidade-Relação
models.py Django ORM 
class Licenciatura(models.Model):
    nome = models.CharField(max_length=200)
    ano_inicio = models.IntegerField()
    descricao = models.TextField(blank=True)

class Docente(models.Model):
    nome = models.CharField(max_length=200)
    area_especializacao = models.CharField(max_length=200, blank=True)
    foto = models.ImageField(upload_to="docentes/", blank=True, null=True)

class UnidadeCurricular(models.Model):
    nome = models.CharField(max_length=200)
    semestre = models.IntegerField()
    licenciatura = models.ForeignKey(Licenciatura, on_delete=models.CASCADE)
    docentes = models.ManyToManyField(Docente, blank=True)

class Tecnologia(models.Model):
    nome = models.CharField(max_length=100)
    descricao = models.TextField(blank=True)
    logo = models.ImageField(upload_to="tecnologias/", blank=True, null=True)

class Projeto(models.Model):
    nome = models.CharField(max_length=200)
    uc = models.ForeignKey(UnidadeCurricular, on_delete=models.CASCADE)
    tecnologias = models.ManyToManyField(Tecnologia, blank=True)

class MakingOf(models.Model):
    entidade = models.CharField(max_length=30, choices=ENTIDADE_CHOICES)
    data = models.DateField(auto_now_add=True)
    descricao = models.TextField(blank=True)
    erros = models.TextField(blank=True)
    correcoes = models.TextField(blank=True)

5. Repositório GitHub

Ver Repositório no GitHub

O Git foi utilizado para controlo de versões ao longo do desenvolvimento, com commits regulares que documentam a evolução do código e permitem reverter para versões anteriores em caso de erro.

O GitHub permitiu manter o código seguro na cloud, acessível a partir do GitHub CodeSpaces. As principais vantagens são: rastreabilidade completa de alterações, colaboração facilitada, e integração com ferramentas de desenvolvimento remoto.
